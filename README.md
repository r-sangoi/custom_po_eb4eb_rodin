# EB4EB Generator & Proof Obligation Annotation for Rodin

## Installation

The Plug-in is still in development so if you want to use it, you will need to configure a plug-in development environmement.

For this, using *Eclipse* :

Follow the instruction to configure [Rodin as a target platform](https://wiki.event-b.org/index.php/Using_Rodin_as_Target_Platform).

Then, you will need [the source of the theory plug-in](https://sourceforge.net/projects/rodin-b-sharp/files/Plugin_Theory/4.0.4/plugins/), each archive will be a different project.

Download the source of this repository containing 2 projects.

Finally, import all of these and [launch Rodin from Eclipse](https://wiki.event-b.org/index.php/Rodin_Developer_Support#2._Launching_Rodin_from_Eclipse), be careful not to unselect the projects you just imported.

## Usage

### EB4EB Generator

Right-clicking a machine display a *Generate EB4EB* button.

![EB4EB_Plugin_Generate_Button](https://github.com/user-attachments/assets/bb0094a5-33a2-4cfe-a33c-d4820ed7d9cf)

Yon can rename the project and context that will be generated or let the default names.

And *Voila*, you have your generated EB4EB context.
